# My Personal Audit 


## 1. Folder Structure

**`blog folder`**: This folder contains blog posts with the file extenion of (.md and .mdx) and other yaml file for the docusarus website blog section
 
**`docs`** : This folder contains the main documentation content for the project displayed in the docs section.               |

**`pages`**: This folder contains a standalone custom pages that is not part of the blog section. 

**`tutorials`**: This folder contains a step-by-step guides that is also separate from the  main docs.

**`src`**: This folder contains the  source code for React components and custom UI logic used in the documentation site.

**`static`**: This folder contains static assets like the images and fonts served directly by Docusaurus.                      |


## 2. My Observations

- There is a clear separation between the docs folder, blog folder and the tutorials folder which actually followed the Docusaurus standard structure.

- **`overview.md` and `README.md`** could cause confusion for beginners.

- The docusaurus config files are well structured 


## 3. Suggestions for Improvement

- I feel like the repository should contain description and a website link to docusarus.

- There should also be a `.template` files for docs, tutorials, and blog posts to maintain formatting and metadata consistency.

- The `contributing.md` file should not be empty, it should contain guidelines on how to contribute to the project, include documentation-specific style rules like Markdown linting, frontmatter requirements and naming conventions.
